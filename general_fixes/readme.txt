Miscellaneous fixes applied to entire database at end of TNRS DB build

Created by: Brad Boyle (bboyle@email.arizona.edu; ojalaquellueva@gmail.com)
Date created: 2013-04-12
Last modified: 2014-03-27

A standalone script of provisional 'hacks' intended to fix issues 
spotted after loading of the database has been completed. Ideally,
a more robust solution should be added to the TNRS pipeline. Until 
that is done, this script should always be run after each database
rebuild.
